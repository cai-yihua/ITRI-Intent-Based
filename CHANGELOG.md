# 版本控制規則

## 命名格式
| **版號格式**                          | **說明**                                                      | **範例**           |
|--------------------------------------|----------------------------------------------------------------|--------------------|
| v[功能]                               | 功能級別。例如：單一意圖、複雜意圖。                             | `v1`               |
| v[功能].[流程]                        | 穩定版。完成一個流程 (如傳送文本、傳送圖片) 後發布。              | `v1.1`              |
| v[功能].[流程]-dev                    | 開發版 (開發流程)。確保「輸入到輸出」流程可運作。                 | `v1.1-dev`         |
| v[功能].[流程].[副版本]-dev           | 整合版 (整合優化)。確保「輸入到輸出，結果正確」。                 | `v1.1.1-dev`       |
| v[功能].[流程].[副版本].[補丁版]-dev  | 補丁版 (修正 Bug)。在前一整合版基礎上修補問題。                   | `v1.1.1.1-dev`     |

## 各欄位含義
1. 功能
    - EX: 單一意圖、複雜意圖
2. 流程
    - EX: 傳送文本、傳送圖片

## 版本遞增原則
1. 版本號從 1 開始遞增
2. 完成一個流程後，在同一個功能內，可以透過補丁版往回修改
3. 完成一個功能後，不往回修改舊版，只能透過 fix 連結版本後修改，並將舊版本加入 `/v[功能]/archive`

## 範例
| **日期**  | **版本**       | **功能 / 流程**    | **修正問題**                |
|-----------|----------------|--------------------|-----------------------------|
| 25/02/01  | v1             | 單一意圖           |                             |
| 25/02/02  | v1.1           | 傳送文本           |                             |
| 25/02/03  | v1.1-dev       | -                  | 開發中，確保文本可送出       |
| 25/02/04  | v1.1.1-dev     | -                  | 整合優化                    |
| 25/02/05  | v1.1.1.1-dev   | -                  | 修補 Bug                    |
| 25/02/06  | v1.2           | 傳送圖片           |                             |
| 25/02/07  | v1.3           | 創建工作流         |                             |
| 25/03/01  | v2             | 複雜意圖           |                             |
| 25/03/02  | v2.2           | 傳送文本           | fix v1.1 格式問題           |
| 25/03/03  | v2.2-dev       | -                  |                           |
